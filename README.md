# Practical lesson pz-UML
Побудова повердінкових UML-діаграм для проєктування інформаційних систем

## Тема - Інформаційно-аналітична система кластерного аналізу для автоматизованого визначення та оцінки загроз у бойових умовах

## 1. Діаграма варіантів використання (Use Case Diagram)

Діаграма відображає межі системи та основні взаємодії між акторами (Аналітиком, Автоматизованими джерелами даних та Командуванням).

`code`

    usecaseDiagram
    actor "Військовий аналітик" as Analyst
    actor "Сенсори / БПЛА" as Sensors
    actor "Система командування" as Command
    
    package "ІАС Кластерного аналізу загроз" {
        usecase "Завантаження даних розвідки" as UC_Data
        usecase "Налаштування параметрів кластеризації" as UC_Config
        usecase "Виконання кластерного аналізу" as UC_Analysis
        usecase "Оцінка ступеня загрози" as UC_Evaluate
        usecase "Візуалізація тактичної обстановки" as UC_View
        usecase "Формування звітів/сповіщень" as UC_Report
    }

    Sensors --> UC_Data
    Analyst --> UC_Data
    Analyst --> UC_Config
    Analyst --> UC_Analysis
    
    UC_Analysis ..> UC_Evaluate : <<include>>
    UC_Evaluate ..> UC_View : <<include>>
    
    UC_View --> Analyst
    UC_Evaluate --> UC_Report
    UC_Report --> Command

`code`

## 2. Діаграма послідовності (Sequence Diagram)

Діаграма демонструє процес обробки даних: від запиту аналітика до отримання результатів оцінки загроз.

`code`

    sequenceDiagram
    autonumber
    actor Analyst as Військовий аналітик
    participant UI as Інтерфейс системи
    participant Engine as Аналітичний модуль (Cluster Engine)
    participant DB as База даних / Сховище
    participant AI as Модуль AI (Оцінка загроз)

    Analyst->>UI: Запит на проведення аналізу
    UI->>DB: Отримання актуальних даних (координати, тип об'єктів)
    DB-->>UI: Набір даних
    UI->>Engine: Передача даних та параметрів (K-means/DBSCAN)
    
    Note over Engine: Групування об'єктів у кластери
    
    Engine->>AI: Передача сформованих кластерів
    AI->>AI: Аналіз щільності та наближення до цілей
    AI-->>Engine: Класифікація загроз (Низька/Висока)
    
    Engine-->>UI: Результати аналізу та координати центроїдів
    UI-->>Analyst: Візуалізація кластерів на карті
    
`code`

## 3. Діаграма діяльності (Activity Diagram)

Діаграма описує логіку роботи алгоритму: від отримання вхідних даних до прийняття рішення про критичну загрозу.

`code`

    activityDiagram
    start
    :Отримання потокових даних з джерел;
    :Попередня обробка та фільтрація шумів;
    
    if (Дані валідні?) then (так)
        :Вибір алгоритму кластеризації;
        :Розподіл об'єктів за кластерами;
        
        partition "Оцінка загрози" {
            :Розрахунок характеристик кластерів;
            :Порівняння з паттернами загроз;
        }
        
        if (Виявлено критичну загрозу?) then (так)
            :Генерація термінового сповіщення;
            :Передача цілевказівок до Системи командування;
        else (ні)
            :Оновлення тактичної карти;
            :Збереження результатів у журнал;
        fi
    else (ні)
        :Логування помилки даних;
    fi
    
    :Формування підсумкового звіту;
    stop
    
`code`

## Опис
* **Use Case Diagram:** Визначає роль аналітика як основного оператора, який керує процесом аналізу, та зовнішніх систем (сенсорів), що постачають дані. Включено критичну залежність оцінки загрози від самого процесу аналізу.

* **Sequence Diagram:** Деталізує взаємодію між внутрішніми компонентами системи. Пояснює, як дані проходять шлях від БД через двигун кластеризації до модуля штучного інтелекту, що визначає пріоритетність цілей.

* **Activity Diagram:** Описує бізнес-логіку системи з розгалуженнями. Особлива увага приділена перевірці валідності даних та прийняттю рішення про негайне реагування у разі виявлення критичних скупчень ворожих сил.
